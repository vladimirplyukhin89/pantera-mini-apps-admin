# Архитектура и стек проекта

Краткое описание того, из чего собран **pantera-mini-apps-admin** и как он вписывается в общую систему.

## Роль в экосистеме

Это **headless CMS**: отдельное приложение с админкой и REST API. Сайт Pantera (фронтенд в другом репозитории) получает контент по HTTP, обращаясь к публичному URL Strapi (например `https://admin.pantera-boxing.ru`).

```
[Браузер пользователя сайта] → статика/SSR фронтенда
                                      |
                                      v
                         HTTPS запросы к Strapi API
                                      |
[Администратор CMS] ──────────→ Strapi Admin (/admin)
```

В этом репозитории **нет** кода основного сайта — только CMS, схемы контента и API.

## Прикладной стек (runtime)

| Слой | Технология |
|------|------------|
| CMS и API | [Strapi](https://strapi.io/) **5.x** (`@strapi/strapi` 5.41.x, Community) |
| Язык бэкенда конфигов и расширений | **TypeScript** (компиляция в JS при сборке) |
| Среда выполнения | **Node.js** 20–24 (см. `engines` в `package.json`) |
| Пакетный менеджер | **npm** |

Админ-панель Strapi построена на **React 18** (зависимости в проекте; сборка идёт через `strapi build`). Это не отдельное SPA-приложение в репозитории — это часть Strapi Admin.

## Плагины и интеграции

| Компонент | Назначение |
|-----------|------------|
| `@strapi/plugin-users-permissions` | Роли, JWT, доступ к API |
| `@strapi/plugin-cloud` | Интеграция со Strapi Cloud (при деплое в облако Strapi) |
| `@strapi/provider-upload-cloudinary` | Загрузка медиа в **Cloudinary** |
| Cloudinary (`CLOUDINARY_*` в `.env`) | Хранение изображений/файлов вне диска сервера |

Конфиг загрузки: `config/plugins.ts`.

## Данные и хранение

| Компонент | Варианты |
|-----------|----------|
| База контента | По умолчанию **SQLite** (`better-sqlite3`), файл `.tmp/data.db` (см. `config/database.ts`) |
| Альтернативы | **PostgreSQL** или **MySQL** через переменные `DATABASE_*` и `DATABASE_CLIENT` |

Локально и на типичном VPS с SQLite база — файл в каталоге приложения; для production с нагрузкой и бэкапами чаще выбирают PostgreSQL.

## Конфигурация и контент

- **`config/`** — `server`, `database`, `admin`, `api`, `middlewares`, `plugins` и т.д.
- **`src/api/`** — типы контента, маршруты, контроллеры и сервисы по доменным сущностям (hero, events, athletes и т.д.).

Публичный URL приложения для корректных ссылок в админке и API задаётся через **`PUBLIC_URL`** (см. `config/server.ts`).

## Деплой (текущая схема на VPS)

Серверная «обвязка» не входит в npm-зависимости, но относится к фактической архитектуре в проде:

| Компонент | Роль |
|-----------|------|
| **Nginx** | TLS-терминация, reverse proxy на Strapi по `127.0.0.1:1337` |
| **Let’s Encrypt (certbot)** | Сертификаты HTTPS |
| **PM2** | Процесс-менеджер для `node` / `npm run start` |
| **UFW** | Файрвол (SSH, 80, 443) |

Подробные команды и порядок действий: [vps-nginx-operations.md](vps-nginx-operations.md).

## Сеть и безопасность

- Доступ к API и админке — по **HTTPS**; секреты (`APP_KEYS`, JWT, salts и др.) только в `.env` на сервере, не в git.
- CORS и публичные права API настраиваются в Strapi (роли Users & Permissions).

## Версионирование и обновления

- Версии Strapi зафиксированы в `package.json`; обновление — через `@strapi/upgrade` или ручное повышение версий с проверкой changelog.
- Фронтенд-сайт собирается с учётом URL API; при смене домена CMS нужно обновить переменные сборки фронта (например `STRAPI_URL`).
