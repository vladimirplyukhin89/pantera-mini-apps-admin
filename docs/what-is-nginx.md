# Что такое Nginx

`Nginx` (читается как "engine-x") — это веб-сервер и reverse proxy.
В продакшне он чаще всего стоит "перед" приложением и принимает входящие HTTP/HTTPS-запросы.

## Зачем он нужен

- Принимает трафик с домена и портов `80/443`.
- Отдаёт SSL/TLS (HTTPS), сертификаты и шифрование.
- Проксирует запросы во внутренние сервисы (например, в Strapi на `127.0.0.1:1337`).
- Делает редиректы (например, `http -> https`).
- Ограничивает размеры тела запроса (`client_max_body_size`), таймауты и буферы.
- Может кэшировать ответы и раздавать статику быстрее приложения.

## Как Nginx связан со Strapi

Типичная схема:

1. Пользователь идёт на `https://admin.pantera-boxing.ru`.
2. Запрос попадает в Nginx.
3. Nginx по `proxy_pass` передаёт запрос в Strapi.
4. Strapi возвращает ответ, Nginx отдаёт его клиенту.

Если Nginx не может получить корректный ответ от Strapi, клиент видит:

- `502 Bad Gateway`
- HTML-страницу ошибки вместо JSON (из-за этого в админке часто появляется `JSON.parse ... unexpected character`).

## Частые причины ошибок при загрузке файлов

- `client_max_body_size` слишком маленький.
- Неправильный `location`/`proxy_pass` для upload endpoint.
- Таймауты слишком короткие для загрузки.
- Upstream (Strapi) перезапускается или падает во время multipart POST.
- Перед сервером есть ещё один прокси/CDN, который меняет поведение запросов.

## Полезные команды диагностики

```bash
nginx -t
nginx -T
systemctl reload nginx
tail -n 200 /var/log/nginx/error.log
```

## Что делают команды и кто их запускает

- `nginx -t` — проверяет синтаксис и валидность конфигов Nginx без перезапуска.
  Запуск: `root` (или через `sudo`).
- `nginx -T` — выводит итоговую объединённую конфигурацию Nginx (включая `sites-enabled`).
  Запуск: `root` (или через `sudo`).
- `systemctl reload nginx` — мягко перечитывает конфиг и применяет изменения без полного даунтайма.
  Запуск: `root` (или через `sudo`).
- `tail -n 200 /var/log/nginx/error.log` — показывает последние ошибки Nginx.
  Запуск: обычно `root` (или через `sudo`), так как лог может быть недоступен обычному пользователю.
- `certbot --nginx -d <domain>` — выпускает/обновляет сертификат и правит Nginx-конфиг автоматически.
  Запуск: `root` (или через `sudo`).
- `certbot certonly --webroot -w <path> -d <domain>` — выпускает сертификат через webroot без автоправок конфига.
  Запуск: `root` (или через `sudo`).
- `pm2 restart pantera-admin --update-env` — перезапускает Strapi-процесс с обновлёнными переменными окружения.
  Запуск: пользователь `deploy` (не `root`), в окружении `nvm`.

### Быстрая памятка по пользователям

- `root`/`sudo`: всё, что связано с Nginx, systemd, сертификатами и `/etc/nginx`.
- `deploy`: всё, что связано с приложением (Strapi), `npm`, `pm2`, `.env` проекта.

## Коротко

`Nginx` — это "входные ворота" для вашего приложения:
он принимает запросы извне, защищает HTTPS и направляет трафик в Strapi.
