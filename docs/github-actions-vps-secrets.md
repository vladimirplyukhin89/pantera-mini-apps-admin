# GitHub Actions: что указывать в VPS_HOST и VPS_USER

Если у вас в панели/доках VPS указано подключение как:

`root@45.153.191.22`

это означает:

- `root` — имя пользователя
- `45.153.191.22` — адрес сервера

## Что писать в Secrets

В GitHub репозитории (`Settings -> Secrets and variables -> Actions`) задайте:

- `VPS_HOST` = `45.153.191.22`
- `VPS_USER` = `deploy` (рекомендуется для деплоя)  
  или `root` (не рекомендуется для регулярного CI-деплоя)
- `VPS_SSH_KEY` = приватный ключ (файл без `.pub`) целиком

Важно: в `VPS_HOST` указывается только IP/домен, **без** `root@`.

## Рекомендуемый вариант для вашего проекта

Так как деплой-команды у вас рассчитаны на пользователя `deploy`, используйте:

- `VPS_HOST` = `45.153.191.22`
- `VPS_USER` = `deploy`

А строка `root@45.153.191.22` нужна только для административного входа под root.

## Быстрая проверка

Локально с вашего ПК:

```bash
ssh -i /home/pirate/.ssh/pantera_github_actions_deploy deploy@45.153.191.22 'whoami'
```

Если выводит `deploy`, значит для `VPS_HOST`/`VPS_USER` значения выбраны правильно.
